Git log에서 특정 path의 변경사항을 제외하고 찾기

$ git log -- ios

위와 같은 명령은 현재 디렉토리 내의 ios라는 디렉토리 내에서 변화가 일어난 커밋의 정보만 출력해주는 명령이다. 그런데 때로는 특정한 디렉토리 내의 변화를 제외하고 보고 싶을 때가 있다. 예를 들어 내가 작업하는 프로젝트 구조는 다음과 같다.

.
├── android
├── ios
├── server1
├── server2
├── server3
└── www

여기서 android와 ios는 각각의 디렉토리에 격리되어 있지만, 서버 프로젝트는 그 외의 모든 디렉토리이다. 실제로는 이것보다 디렉토리가 더 많고 심지어 루트에도 관련 파일이 산재되어 있다. 이런 경우 ios와 android 프로젝트의 변화는 다음과 같이 해당 디렉토리를 검색하면 되지만

$ git log -- ios
$ git log -- android

서버 프로젝트의 변화를 알기 위해서는 어떻게 해야 할까? ios와 android 내의 변화를 제외하고 찾아야 한다. 이럴 때 쓸 수 있는 방법이다. pathspec magic이라는 기능이 있다고 한다. pathspec을 지정할 때 prefix로 :(exclude)을 붙이는 것이다. 단축표현으로 :!나 :^을 쓸 수도 있다. (나 ! 같은 특수문자는 shell에서 메타문자로 쓰이는 것들이므로 shell에 의해 다른 값으로 해석되지 않고 git까지 전달하기 위해서는 이 값을 quoting하여 감싸야 한다. :나 ^의 경우 bash의 expansion에 쓰이지 않으므로 quote 없이 사용할 수 있다. (실제로 ^는 quote 없이 사용할 수 있게 하기 위해 나중에 추가된 것이라고 한다.

$ git log -- ':(exclude)ios'
$ git log -- ':!android'
$ git log -- ':!ios' ':!android'
$ git log -- :^ios :^android

"..."로 감싸는 경우 bash의 expansion을 거칠 수 있으므로 '...' 가 안전하다.
exclude matching은 디렉토리 뿐 아니라 path 전체에 대해서 할 수 있으며 * 같은 와일드카드도 사용 가능하다.
exclude 뒤에 옵션도 붙일 수 있는 듯하다. 예를 들어 대소문자 상관 없이 매칭시키려면 icase를 추가한다.

$ git log -- ':(exclude,icase)iOS'
$ git log -- ':(exclude,glob)sub/*/file'

위에서 glob의 역할은 정확히 모르겠다..

func getAllPerson() -> [NSManagedObject]? {
    let fetchRequest = NSFetchRequest(entityName: "Person")
    do {
        let results = try self.managedObjectContext.executeFetchRequest(fetchRequest)
        let people = results as! [NSManagedObject]
        return people
    } catch let error as NSError {
        print("Could not fetch \(error), \(error.userInfo)")
        return nil
    }
}
